# 🔥 BurnerChat

> 焚後即毀、端對端加密的團隊即時聊天室。訊息在你設定的秒數後自動燒掉,伺服器只看得到密文。

專為「不能用 Slack、又不想留訊息紀錄」的情境設計。開一次伺服器、貼房間連結給同事、聊完直接關掉。訊息不落地、不寫 log、重開 server 一切歸零。

## 🔥 它是什麼

BurnerChat 由兩個東西組成:

- **Server** — 一個 Node.js 腳本,在你的機器上跑一個聊天伺服器。房間資料全部只在記憶體,關掉就全消失。
- **Client** — 使用者從瀏覽器打開房間網址就能用,不需要裝任何東西。另外也提供 VSCode 擴充套件讓工程師在編輯器裡聊天。

**三個設計取向你應該先知道**:

1. **不是雲端服務**,是你自己跑在 LAN 或公司內網裡的小伺服器
2. **沒有帳號**,只有房間 + 房間密碼
3. **沒有訊息歷史**,訊息是流動的、看完會燒掉,Server 重開一切歸零

### 主要功能

| 功能 | 說明 |
|------|------|
| 🔒 **端對端加密** | AES-256-GCM,金鑰從房間密碼派生 (PBKDF2-SHA256)。Server 只看得到密文。 |
| 🔥 **訊息焚毀** | 可設 0-3600 秒倒數,倒數完訊息自動消失(`0` = 不燒) |
| 👁 **已讀回執** | 知道誰讀過你的訊息 |
| 🏠 **多房間** | Admin 可以隨時建新房間,每個房間有獨立密碼 |
| 📋 **一鍵複製邀請** | Admin 後台點一下,把「房間連結+密碼」複製好給你貼 Slack/LINE |
| 🔔 **音效提醒** | Teams 風格的兩音符提示音,可關 |
| 📊 **分頁未讀數** | 標題列顯示 `(3) BurnerChat`,分頁切回自動歸零 |
| 🌐 **HTTP 長輪詢** | 不使用 WebSocket,在企業防火牆下也能用 |

# 📦 Part A — 架設者指南

這段寫給會拿到 `burner_chat_installer.py` 的你。跑完你會得到一個聊天伺服器 + 房間網址 + 密碼,可以貼給同事開始聊。

## A1. 系統需求

- **Python 3.8+**
- **Node.js 18+**
- 網路上其他人**能連到你這台機器**(通常是同個 LAN 或 VPN)
- (選配) **VSCode** — 若有安裝,會順便裝聊天室 extension;沒有的話 installer 會自動跳過這步

## A2. 三分鐘快速開始

```bash
python3 burner_chat_installer.py
```

跑完會看到:

```
╔══════════════════════════════════════════════╗
  🔥  BurnerChat 已就緒!
╚══════════════════════════════════════════════╝

  📡 預設房間網址:
  http://192.168.1.100:7788/room/a1b2c3d4
  🔑 預設房間密碼: xK3pQ9vN2mRs

  ╭───────────── 🛡️  ADMIN 後台 ─────────────╮
  │ 後台網址: http://192.168.1.100:7788/admin
  │ 管理員密碼: Zy7F3kP2xM9nRbV8qCwLuH
  │ (此密碼僅此處可見,不會顯示在網頁上)
  ╰────────────────────────────────────────────╯

  按 Ctrl+C 關閉伺服器並銷毀所有訊息
```

**接著做兩件事**:

1. 把「預設房間網址」和「預設房間密碼」複製給要聊天的人
2. 把「管理員密碼」記在安全的地方(之後開新房間要用),**不要**把它外流

**完成了。** 之後要關伺服器、銷毀所有訊息,按 Ctrl+C。

## A3. CLI 參數

```bash
python3 burner_chat_installer.py                   # 預設啟動(自動找可用 port)
python3 burner_chat_installer.py --port 80         # 指定 port
python3 burner_chat_installer.py --marketplace     # 產生可上架 VSCode Marketplace 的套件
python3 burner_chat_installer.py --help            # 顯示說明
```

## A4. Windows + Port 80(內網政策要求時)

如果你的網路政策只開放 port 80:

```powershell
# 用「系統管理員身分」開 PowerShell,然後:
python burner_chat_installer.py --port 80
```

### 注意事項

- 必須是管理員權限,否則 node.exe 無法綁 port 80
- 第一次啟動時,Windows 防火牆會問要不要允許 node.exe 存取網路 — 按「允許」
- 若遇到 port 占用錯誤,通常是 IIS 在搶:`iisreset /stop`
- 若遇到 Skype 占用,關掉或到「設定 → 進階」取消「使用 port 80/443」

## A5. Linux / Mac + Port 80

```bash
sudo python3 burner_chat_installer.py --port 80
```

或者授權 node 一次(之後就不用每次 sudo):

```bash
sudo setcap 'cap_net_bind_service=+ep' $(which node)
python3 burner_chat_installer.py --port 80
```

## A6. Admin 後台操作

Server 啟動時 console 會印出 admin 密碼(**只此一次**),記下來。

### 進入後台

打開 `http://你的IP/admin`(port 80)或 `http://你的IP:7788/admin`(預設 port),輸入管理員密碼。

### 可以做的事

| 操作 | 說明 |
|------|------|
| 📋 **看所有房間** | 列出每個房間的名稱、密碼、線上人數、焚毀秒數 |
| ➕ **建立新房間** | 輸入名稱(例如「產品會議」、「Code Review Q4」),自動產生 12 位隨機密碼 |
| 🔗 **一鍵複製邀請** | 點房間旁邊的「複製」,剪貼簿就有格式化文字,直接貼 Slack/LINE/Email |

### 複製的邀請文字長這樣

```
🔥 產品會議
🔗 http://10.0.0.5/room/b2c3d4e5
🔑 xK3pQ9vN2mRs
```

### Admin 看得到訊息內容嗎?

**看不到**。Admin 看得到房間**存在**、名稱、密碼、有多少人在線,但**訊息內容是端對端加密的**,admin 後台跟 server 本身一樣只看得到密文。

## A7. 停止伺服器

在執行 installer 的終端機按 **Ctrl+C**。

訊息、房間、密碼、tokens **全部**在記憶體,按下去就灰飛煙滅。

## A8. 進階:上架 VSCode Marketplace

如果你要把 client 上架到 marketplace 讓更多人使用:

```bash
python3 burner_chat_installer.py --marketplace
```

會在 `~/.burner-chat/marketplace/` 產生上架用的套件目錄(含你指定的 publisher、repo、license、icon、README、CHANGELOG)。

之後照 script 最後印出的步驟操作:
1. `npm install -g @vscode/vsce`
2. 到 `dev.azure.com` 建 Personal Access Token
3. `marketplace.visualstudio.com/manage` 建 publisher
4. `vsce login <publisher>` → `vsce package` → `vsce publish`

# 💬 Part B — 使用者指南

這段寫給「從別人手上拿到房間網址和密碼」的你。不用裝任何東西,只要一個瀏覽器。

## B1. 加入房間

1. 從朋友/主管拿到房間連結(類似 `http://10.0.0.5/room/a1b2c3d4`)和密碼
2. 打開連結,輸入你的暱稱和密碼
3. 開始聊

**暱稱**可以是任何 20 字內的名字,每次進房都能改,不用註冊。

## B2. 發送訊息

- 輸入訊息,按 **Enter** 送出
- 支援中文、表情符號、特殊字元
- 訊息會在倒數秒數後自動燒掉

## B3. 訊息焚毀時間

每則訊息送出後會顯示倒數(`🔥 28s`),到時間自動消失。

預設 30 秒。房間上方「焚後:」欄位可以改成:

| 秒數 | 效果 |
|------|------|
| `0` | 永不燃燒(直到重新整理頁面) |
| `30` | 30 秒後燒掉(預設) |
| `60` / `300` / `3600` | 其他秒數,最多 1 小時 |

> ⚠️ **這設定是「整個房間共用」** — 你改完其他人也會受影響,會有系統訊息通知。

## B4. 已讀回執

你發送的訊息下方會顯示:

```
✓ 已讀: 小明, 小華
```

超過 3 人讀過會顯示 `+N`(例如 `✓ 已讀: 小明, 小華, 小強 +2`)。

## B5. 音效與未讀數

- **🔔 提示音** — 右上角可開關,新訊息進來時播短提示音
- **分頁未讀數** — 切到別的分頁時,有人發訊息會讓標題變成 `(3) 🔥 BurnerChat`,切回來自動歸零

## B6. 離開房間

關分頁就好。Server 會在 60 秒後把你標記為離線,其他人看得到「XXX 已離開」的系統訊息。

## B7. VSCode 擴充套件(給工程師)

如果架設者有裝 VSCode extension 給你,或你從 marketplace 裝了版本:

- 按 `Ctrl+Shift+B`(Mac: `Cmd+Shift+B`)
- 或從命令面板搜尋 `BurnerChat: Connect`
- 貼上房間網址、輸入暱稱密碼,在編輯器裡就能聊

# 🔐 安全模型

(這段架設者和使用者都應該讀。)

## ✅ 保護你免於

- **LAN 封包側錄** — 同 Wi-Fi 下用 Wireshark 抓也只看到亂碼
- **MITM 中間人** — 伺服器被換掉也看不到內容(因為密鑰在雙方腦袋裡)
- **伺服器被 dump 記憶體** — Server 根本沒拿到明文
- **事後錄製的流量重播** — 密文加上 nonce,無法 replay

## ❌ 不保護你免於

- **知道房間密碼的人** — 密碼就是鑰匙,洩漏等於房間破功
- **端點被入侵** — 對方的筆電被控制、螢幕被錄了,加密幫不上忙
- **流量分析** — Admin 和 Server 看得到:誰在線、誰跟誰同房、訊息何時發
- **取證分析** — 瀏覽器的記憶體 dump 可能撈到訊息內容

## 建議的使用習慣

1. **密碼要長** — 用預設產生的 12 字隨機密碼,不要改成 `1234`
2. **分享管道要隔離** — 密碼別貼到會被記錄的地方
3. **敏感訊息用短焚毀時間** — 10-30 秒夠用
4. **聊完關 Server** — Ctrl+C 讓所有訊息立刻蒸發
5. **別重用密碼** — 每個房間用新的

# ❓ 常見問題

### Q: WebSocket 被公司網路擋了怎麼辦?

**不是問題**。BurnerChat 從 v1.5 開始改用 HTTP 長輪詢,只走 GET / POST,任何能上 HTTP 的網路都能用。

### Q: 訊息會不會被存到磁碟?

**不會**。Server 所有資料只在記憶體,按 Ctrl+C 或重啟機器就全沒了。

### Q: 瀏覽器紀錄會留下訊息嗎?

瀏覽器本身不會主動存聊天訊息,但**記憶體 dump 或取證工具**仍可能撈到剛顯示過的內容。真的敏感請聊完關分頁。

### Q: 可以跨公網用嗎?

技術上可以,但**強烈建議在前面加 HTTPS**(用 nginx/Caddy)。目前 BurnerChat 本身不做 TLS。

### Q: 為什麼我用 `http://192.168.x.x` 看訊息卻加密失敗?

瀏覽器的 `crypto.subtle` API 預設只在 `https://` 或 `localhost` 下啟用。BurnerChat 偵測到這情況會自動切到**純 JavaScript 版本**的 AES,**登入時多花 2-3 秒**派生金鑰(localhost 則瞬間)。功能完全相同。

### Q: 多少人同時用沒問題?

一般電腦上,單一房間**數十人**沒問題。更大規模沒測過,但不是它的設計情境。

### Q: 訊息有字數上限嗎?

單則密文最多 14 KB,對應約 7-10 KB 明文。一般對話綽綽有餘,超過會被 server 拒絕。

### Q: 支援檔案傳輸嗎?

**不支援,刻意的**。焚後即毀的概念下,檔案會複雜化。要傳檔請用別的工具(記得先加密)。

### Q: 支援語音 / 視訊嗎?

**不支援**。這是純文字聊天工具。

### Q: 可以永久保留某些訊息嗎?

把焚毀時間設 `0` 可以讓訊息不自動燒,但重新整理頁面還是會消失,Server 重啟也會消失。**沒有「永久儲存」的概念**,這是刻意的。

### Q: 斷網重連後訊息會補回來嗎?

**會,但有限**。Server 記憶體保留最近 200 筆事件;短暫斷線重連會自動補回中間漏掉的。但斷線太久(或事件超過 200 筆)就拿不回來了。

# ⚙️ 技術規格

給想了解內部的人看。

| 項目 | 規格 |
|------|------|
| **加密** | AES-256-GCM,每則訊息獨立 96-bit IV |
| **金鑰派生** | PBKDF2-HMAC-SHA256,200k 迭代(WebCrypto)或 50k(純 JS fallback) |
| **Salt** | `burnerchat-v1:<roomId>`(綁定房間 ID,跨房間不共用金鑰) |
| **傳輸** | HTTP long polling(GET `/poll` hold 最多 25 秒) |
| **訊息長度上限** | 14 KB 密文(約 7-10 KB 明文) |
| **Token 有效期** | 認證後 1 小時 |
| **離線判定** | 60 秒無活動視為離線(每 15 秒掃一次) |
| **訊息 log** | Server 記憶體保留最近 200 筆(僅用於短暫重連補追) |
